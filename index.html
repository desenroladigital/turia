Modelos & Operação

Dois níveis, um cockpit: Base para volume barato e Reforço para o trabalho pesado — protegido por senha.

🧠 Cérebro (Gateway iaCentral) — teste isolado

Liga o chat direto na função iaCentral da ponte: Prompt Matriz, base de conhecimento e roteamento rodam no SERVIDOR (namespace ia/). Sistema e GuiD ficam intocados.

No modo cérebro, as páginas locais (Matriz/Conhecimento) NÃO são enviadas — valem as do servidor. Desligue o modo para voltar às conexões diretas.

Conexão · Provedor aberto (OpenAI-compatível)

DeepInfra, OpenRouter, Together, Groq… basta a URL base e a chave.

Conexão · Anthropic (opcional, premium)

Camada extra para casos extremos. Na pré-visualização do Claude funciona sem chave (demonstração).

Roteamento por tarefa (multiservidor)

A ordem manda: o cérebro tenta o 1º e cai para o próximo (failover). Formato apelido:modelo, um por linha. Salvar já vale na próxima resposta — sem deploy.

Cérebro consciente · propostas de troca

Lê o uso real (custo, latência, erro) e propõe trocas de modelo por tarefa. Nada muda sozinho — você aprova.

Memória & Conhecimento (porta do KOS)

A base que os agentes consultam. O KOS entra por aqui: importe o JSON exportado dele para alimentar a memória global do cérebro. Nada é removido da origem — só copiado.

—
Formato aceito: array de registros ou { registros: [...] }, com campos como titulo/conteudo/categoria/tags. Quando o KOS estiver pronto, é só apontar o export dele aqui.
